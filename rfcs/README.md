# Mobazha Requests for Comment

RFCs are public proposals for material changes to protocol, policy, economics,
security, governance, or cross-repository ownership. An RFC creates a review
record; it does not activate behavior.

## Lifecycle

`Draft → Review → Accepted/Rejected/Withdrawn → Implemented or Superseded`

Accepted means the proposal was approved. Implemented additionally requires
identified release evidence. Draft, Review, and Accepted RFCs must not be
described as shipped capability.
